CPP没有`interface`关键字,所有的接口和抽象类一样，采用`class`定义，例如这里由一个抽象的接口`Say`，如下：

```cpp
class Say
{
public:
    virtual say();
};

虚函数

`virtual`表示的是该函数是虚函数，虚函数最重要的用处是：基类可以调用子类的具体实现。虚函数有一个兄弟叫纯虚函数，如下：

class Say
{
public:
    virtual say()=0;
};

当一个函数用`virtual`标记，并且函数结尾加上`=0`则表示该函数为纯虚函数，纯虚函数的功能是被标记的类必须由子类实现，本身不可以被实例化。

规范

  • 所有的文件均以小写命名，不同单词直接使用`_`连接，例如`my_class.cpp`。
  • 类的定义使用驼峰命名的方式，首字母大写，例如`MyClass`。
  • 头文件需要加上`ifndef`、`define`、`endif`进行包装，定义的宏命名需要遵循`项目名_模块名_文件名_H`的格式。
#ifndef PROJECT_MODULE_FILE_H
#define PROJECT_MODULE_FILE_H
//code
#endif
